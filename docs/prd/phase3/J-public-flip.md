## 규모 산정

- **S (Small)** — 코드 작업은 스캔·검증 자동화 정도. 공개 자체는 비가역 human 스위치.
- v1의 D(미러 자동화)+F(미러 공개)를 대체·통합한다 — **상시 미러 인프라가 사라진 것이 v2의 핵심 절감**: 비밀 코드가 없으므로 "리포 그 자체"가 공개 대상이고, filter-repo는 스캔 실패 시 1회성 도구로 강등된다.

## 요구사항

본 리포(또는 1회 필터링된 사본)를 외부 공개하고, 4개 호스트의 설치 경로가 공개 리포에서 동작함을 검증한다.

## D4 — 히스토리 처분 (스캔 결과가 결정)

> ✅ **해소 (2026-06-12)**: 전수 스캔 2종 + 수동 감사 **전부 클린** → **본 리포 직접 공개(filter-repo 불요)** 확정.
> - gitleaks v8 (podman, 전체 히스토리 189 커밋): leaks 0
> - trufflehog v3.95.5 (전체 히스토리, 7260 chunks): verified 0 · unverified 0
> - 수동 감사: JWT형 문자열(`eyJ…`) 히스토리 grep 0건
> - 공개 **실행**은 여전히 I(라이선스) 선행 — 아래 "I 전 공개 금지" 규칙 불변. 공개 직전 재스캔 1회 권장(이후 커밋 커버).

1. **시크릿 전수 스캔**: gitleaks + trufflehog를 전체 히스토리에 (현재·과거 커밋 전부). 대상: 자격증명·토큰·내부 URL 중 비공개 유지가 필요한 것. *정책·코드 평문은 v2에서 스캔 대상이 아니다* (D2 채택: 비밀 아님).
2. **통과** → 본 리포 직접 공개(미러·filter-repo 불요). ← **이 분기로 확정.**
3. **실패** → 해당 경로만 `git filter-repo` 1회 + 자격증명 로테이션 + 재스캔. (v1 D의 "비공개 코드 영구 노출" 공포는 "시크릿 노출"로 축소된다.)
